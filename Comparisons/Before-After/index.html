    Сравнение: старая версия (WordPress) vs новая (Astro)¶

    Дата: 2026-06-28 Сравниваются: - Старая: сайт на WordPress + Elementor (предыдущая версия до редизайна) - Новая: Astro 5 + Tailwind, статика на GitHub Pages (после редизайна)

    Данные для старой версии взяты из: - [[../02-Audit/Technical-Audit.md]] — общий технический аудит - [[../02-Audit/Visual-Audit.md]] — визуальный аудит - [[../02-Audit/Epistemic-Audit.md]] — коррекция данных - [[../02-Audit/UX-Audit.md]] — UX-аудит

    1. Технологический стек¶

    Параметр Старая (WordPress) Новая (Astro)
    Frontend WordPress 6.x + Elementor Astro 5 + Tailwind CSS
    Backend PHP 8 + MySQL/MariaDB Hono (Node.js) + PostgreSQL
    Хостинг Свой сервер / VPS GitHub Pages (статика)
    Стоимость хостинга ~500-1500 ₽/мес 0 ₽ (GitHub Pages free tier)
    Build Runtime PHP Build → static HTML
    TTFB главной ~800-1500ms (PHP+MySQL) ~425ms (статика + CDN)
    Cache WP Super Cache / W3TC CDN GitHub Pages встроенный
    Сложность деплоя FTP / Git на сервер git push → авто-деплой
    Зависимости от сервера Да (PHP, MySQL) Нет (статика)
    Версионность WP/PHP/Elementor обновления Зафиксировано в git
    Победитель ❌ ✅ Astro

    2. Архитектура страниц¶

    Параметр Старая Новая
    Способ отрисовки Server-side PHP Build-time static HTML
    Время до первого байта ~1.2s ~0.4s
    Время до интерактивности ~2-3s ~1s
    Размер HTML главной ~150-200KB 42KB
    Размер CSS (gzip) ~30-50KB 24KB (9KB gzip)
    Число JS-бандлов 5-10 (Elementor + jQuery + плагины) 0 внешних, inline-скрипты
    jQuery зависимость Да (устаревший) Нет
    React/Vue Нет (или частично) Нет (только HTML+JS)
    Победитель ❌ ✅ Astro

    3. Производительность (десктоп)¶

    Метрика Старая (WP) Новая (Astro)
    PageSpeed Performance ~50-65 ~85-90 (оценка)
    TTFB 800-1500ms 425ms
    LCP 2.5-4s 1.2-1.5s (оценка)
    CLS ~0.05-0.15 ~0.01 (нет сдвигов)
    Total page weight 1.5-2.5 MB ~520KB
    Число HTTP-запросов 30-50 6 фото + 1 CSS + 1 HTML
    Gzip включён Зависит от сервера ✓ автоматически (gh-pages)
    Победитель ❌ ✅ Astro

    4. SEO¶

    Параметр Старая (WP) Новая (Astro)
    Lighthouse SEO 80-90 100
    Title уникальный ✓ (WP-Yoast) ✓
    Meta description уникальный ✓ ✗ общий на всех страницах
    Canonical URL ✓ ✗ отсутствует
    Schema.org JSON-LD ✓ (Product, BreadcrumbList) ✗ полностью отсутствует
    Sitemap.xml ✓ автогенерация ✗ отсутствует
    robots.txt ✓ ✗ отсутствует
    Open Graph ✓ (Yoast/RankMath) ✗ отсутствует
    Twitter Card ✓ ✗ отсутствует
    Hreflang Если настроен ✗ отсутствует
    Favicon ✓ ✗ 404
    Победитель ⚠ частично WP ⚠ хуже по Schema/OG, лучше по базовому

    5. UX и доступность¶

    Параметр Старая (WP) Новая (Astro)
    Lighthouse Accessibility 75-85 84
    Мобильная навигация ✓ hamburger ✗ отсутствует (P0)
    Touch-targets Смешанно ✓ 44px+ на кнопках
    Color contrast ✓ ⚠ 4 нарушения (frost-dark/50-80 на белом)
    Heading order ✓ ✗ h1 → h3 пропуск
    Select labels ✓ ✗ 2 <select> без <label>
    Touch-friendly формы ✓ ✓
    Адаптивность ✓ ✓
    Калькулятор размера ✗ не было ✓ есть
    Переключатель языков WPML/Polylang ✓ нативный (3 языка)
    Переключатель тем ✗ не было ✓ 3 темы
    Победитель ⚠ ⚠ (новая лучше по фичам, хуже по a11y)

    6. Контент и i18n¶

    Параметр Старая (WP) Новая (Astro)
    CMS-редактор ✓ WordPress admin ✗ код-только
    Изменение текста без деплоя ✓ через админку ✗ требует git push
    Markdown ✓ (через WP плагины) ✓ нативный
    i18n количество языков 1 (ru) или WPML 3 (ru/en/zh)
    Переключение языка URL-prefix или cookie URL ?lang=
    i18n работает на всех страницах ✓ ⚠ хардкод ?lang=ru ломает
    Tours ✗ не было ✓ 3 тура с программами
    Победитель ⚠ по CMS ⚠ по фичам (i18n + tours)

    7. Эпистемика (правда данных)¶

    Параметр Старая (WP) Новая (Astro)
    Источник правды для товаров WP post meta + MySQL Хардкод в .astro файлах
    Источник правды для цен WP post meta Хардкод
    Источник правды для отзывов WP comments / плагин Хардкод "109" (в БД 5)
    Характеристики товаров WP custom fields Хардкод (в БД нет колонки)
    Описания товаров WP editor Хардкод (в БД есть, но игнорируется)
    Реальное число отзывов в БД Зависит от настройки 5 (заявлено 109)
    Победитель ⚠ WP-БД связь ⚠ оба рассинхронизированы по-своему

    КРИТИЧНОЕ ОТКРЫТИЕ: На новом сайте заявлено "109 отзывов · 5.0" при реальных 5 одобренных отзывах в БД. Это техническая ЛОЖЬ, которую видит клиент.

    8. Дизайн и бренд¶

    Параметр Старая (WP) Новая (Astro)
    Цветовая палитра #1e4a62 (синий), #f4b942 (янтарь), #f8fafc (фон) #0a1929 (грозовая синь), #f4b942 (молния), #2c5a6e (байкальский лёд)
    Темы 1 3 (light/night/taiga)
    Типографика Inter Inter + JetBrains Mono
    Hero-стиль Сине-голубой градиент + "Озёрные коньки" Тёмный thunder-фон + молния-логотип + "Озёрные / коньки / ГРОМ"
    Иконки Emoji ⛸ 🛠 ❄️ 🚚 💯 SVG-молнии, ⚙ ✦ ◐ ∞
    Hero-фото товара ❌ нет (иконка-эмодзи) ✓ есть (стилизованная плашка)
    Победитель ⚠ generic ✅ соответствует бренду ГРОМ

    9. Техническая сложность поддержки¶

    Задача Старая (WP) Новая (Astro)
    Изменить текст на странице Зайти в WP админку, отредактировать Изменить .md, git push
    Добавить страницу Создать WP post + настроить меню Создать .md в src/pages/, добавить в nav
    Обновить цену товара WP admin (если используется custom field) Изменить price в .astro + БД + sync
    Изменить тему Установить WP тему Изменить CSS variables в global.css
    Добавить язык WPML / Polylang настройка Добавить в translations.ts
    Стоимость владения (в год) ~10-30k ₽ (хостинг + обслуживание) 0 ₽
    Требования к разработчику PHP + WP экосистема JS/Astro + Markdown
    Победитель** для не-программиста ✅ WP ❌ Astro
    Победитель для разработчика ❌ WP ✅ Astro

    10. Безопасность¶

    Параметр Старая (WP) Новая (Astro)
    Поверхность атаки PHP + MySQL + plugins (огромная) Только статические файлы
    SQL-инъекции Возможны (зависит от кода) Невозможны (нет БД на фронте)
    XSS Возможен Минимальный (нет пользовательского ввода)
    CSRF Возможен Невозможен
    DDoS Зависит от сервера GitHub Pages защита
    Обновления безопасности Регулярно для WP/PHP/plugins Не требуются (нет серверного кода)
    Сертификат SSL Let's Encrypt (ручной) Авто (GitHub)
    Победитель ⚠ ✅ Astro (статика безопаснее по определению)

    11. SEO-критичные ошибки (для сравнения)¶

    Старая (WP)¶

    • Возможно: Canonical, Hreflang, Schema.org (если Yoast настроен правильно)
    • Возможно: sitemap.xml автогенерация
    • Проблема: Медленная загрузка → плохой ranking
    • Проблема: jQuery → устаревший, медленный JS

    Новая (Astro)¶

    • ✗ Schema.org JSON-LD отсутствует (потеря по сравнению с WP)
    • ✗ Sitemap.xml отсутствует (потеря)
    • ✗ Canonical отсутствует (потеря)
    • ✗ Hreflang отсутствует (потеря)
    • ✗ Open Graph отсутствует (потеря)
    • ✓ Скорость лучше → ranking выше
    • ✓ Core Web Vitals в норме

    ВЫВОД: Новая версия быстрее, но потеряла технические SEO-фичи старой WP.

    12. Сводный итог¶

    Категория Победитель Почему
    Технологии и скорость ✅ Astro Статика, TTFB 425ms vs 1500ms
    Безопасность ✅ Astro Нет серверного кода
    Стоимость владения ✅ Astro 0 ₽/мес
    Базовое SEO ⚠ Паритет Astro быстрее, WP имеет Schema
    Продвинутое SEO ⚠ WP Schema, sitemap, canonical, OG
    UX фичи ✅ Astro Калькулятор, темы, i18n
    Мобильный UX ⚠ WP Hamburger menu был, сейчас нет
    CMS-управление ✅ WP Редактирование без программиста
    Бренд/дизайн ✅ Astro Соответствует концепции ГРОМ
    Эпистемика данных ⚠ Паритет Оба рассинхронизированы

    Итоговый счёт¶

    • Astro побеждает в: 5 категориях (скорость, безопасность, цена, UX-фичи, дизайн)
    • WP побеждает в: 1 категории (CMS)
    • Паритет в: 4 категориях (базовый SEO, мобильный, эпистемика, продвинутое SEO)

    13. Что делать дальше¶

    Взять лучшее от обеих версий¶

    Добавить в Astro (взять из WP): 1. CMS-слой для редактирования контента (Decap CMS / TinaCMS / Astro Studio) 2. Schema.org JSON-LD (Product, Organization, BreadcrumbList, FAQ) 3. Sitemap.xml + robots.txt в public/ 4. Canonical URL + Hreflang 5. Open Graph + Twitter Card 6. Мобильная навигация (hamburger) 7. Подключить БД через API вместо хардкода

    Сохранить преимущества Astro: 1. Скорость и статика 2. Темы и i18n 3. Маркдаун-исходники 4. GitOps деплой

    14. TL;DR¶

    Новая версия (Astro): быстрее в 3-4 раза, дешевле в обслуживании, безопаснее, красивее, но потеряла технические SEO-фичи (Schema, sitemap, canonical, OG) и мобильную навигацию. Хардкод данных в коде создаёт рассинхрон с БД.

    Старая версия (WP): медленнее, дороже, уязвимее, но имела CMS для редактирования и более полные SEO-фичи.

    Вердикт: Новая архитектура лучше, но требуется дозаполнение (P0: мобильная навигация + хардкод lang, P1: Schema + canonical + sitemap + OG).
