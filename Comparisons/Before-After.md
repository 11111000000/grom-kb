# Сравнение: старая версия (WordPress) vs новая (Astro)

**Дата:** 2026-06-28
**Сравниваются:**
- **Старая:** сайт на WordPress + Elementor (предыдущая версия до редизайна)
- **Новая:** Astro 5 + Tailwind, статика на GitHub Pages (после редизайна)

> Данные для старой версии взяты из:

## 1. Технологический стек

| Параметр | Старая (WordPress) | Новая (Astro) |
|---|---|---|
| **Frontend** | WordPress 6.x + Elementor | Astro 5 + Tailwind CSS |
| **Backend** | PHP 8 + MySQL/MariaDB | Hono (Node.js) + PostgreSQL |
| **Хостинг** | Свой сервер / VPS | GitHub Pages (статика) |
| **Стоимость хостинга** | ~500-1500 ₽/мес | **0 ₽** (GitHub Pages free tier) |
| **Build** | Runtime PHP | Build → static HTML |
| **TTFB главной** | ~800-1500ms (PHP+MySQL) | **~425ms** (статика + CDN) |
| **Cache** | WP Super Cache / W3TC | CDN GitHub Pages встроенный |
| **Сложность деплоя** | FTP / Git на сервер | `git push` → авто-деплой |
| **Зависимости от сервера** | Да (PHP, MySQL) | **Нет** (статика) |
| **Версионность** | WP/PHP/Elementor обновления | Зафиксировано в git |
| **Победитель** | ❌ | ✅ Astro |

## 11. SEO-критичные ошибки (для сравнения)

### Старая (WP)
- **Возможно:** Canonical, Hreflang, Schema.org (если Yoast настроен правильно)
- **Возможно:** sitemap.xml автогенерация
- **Проблема:** Медленная загрузка → плохой ranking
- **Проблема:** jQuery → устаревший, медленный JS

### Новая (Astro)
- **✗ Schema.org JSON-LD отсутствует** (потеря по сравнению с WP)
- **✗ Sitemap.xml отсутствует** (потеря)
- **✗ Canonical отсутствует** (потеря)
- **✗ Hreflang отсутствует** (потеря)
- **✗ Open Graph отсутствует** (потеря)
- **✓ Скорость лучше** → ranking выше
- **✓ Core Web Vitals в норме**

**ВЫВОД:** Новая версия быстрее, но потеряла технические SEO-фичи старой WP.

## 13. Что делать дальше

### Взять лучшее от обеих версий

**Добавить в Astro (взять из WP):**
1. CMS-слой для редактирования контента (Decap CMS / TinaCMS / Astro Studio)
2. Schema.org JSON-LD (Product, Organization, BreadcrumbList, FAQ)
3. Sitemap.xml + robots.txt в `public/`
4. Canonical URL + Hreflang
5. Open Graph + Twitter Card
6. Мобильная навигация (hamburger)
7. Подключить БД через API вместо хардкода

**Сохранить преимущества Astro:**
1. Скорость и статика
2. Темы и i18n
3. Маркдаун-исходники
4. GitOps деплой

## 14. TL;DR

> **Новая версия (Astro):** быстрее в 3-4 раза, дешевле в обслуживании, безопаснее, красивее, но потеряла технические SEO-фичи (Schema, sitemap, canonical, OG) и мобильную навигацию. Хардкод данных в коде создаёт рассинхрон с БД.
>
> **Старая версия (WP):** медленнее, дороже, уязвимее, но имела CMS для редактирования и более полные SEO-фичи.
>
> **Вердикт:** Новая архитектура лучше, но требуется дозаполнение (P0: мобильная навигация + хардкод lang, P1: Schema + canonical + sitemap + OG).
