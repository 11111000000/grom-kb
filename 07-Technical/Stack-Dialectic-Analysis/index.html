    🧠 Диалектический анализ стека ГРОМ38¶

    Цель: выбрать максимально современный и гармоничный стек для новой версии сайта, объединяющей магазин (озёрные коньки) и туристическое агентство (БайкалТур), с управлением через AI-агента.

    🎯 Методология¶

    Применяю диалектический метод (Гегель): 1. Тезис — что есть сейчас (текущий стек) 2. Антитезис — что не работает, какие ограничения 3. Синтез — каким должен быть новый стек

    12. Агент управляет турагентством

    14. Агент общается с клиентами

    🚫 АНТИТЕЗИС — что не работает, какие ограничения¶

    1. Проблемы текущего стека (по эпистемическому аудиту)¶

    Проблема Влияние Подтверждено
    26 багов, из них 9 P0 Блокирует продажи ✅
    Главная пустая (0 H1, 0 текста) -70% вовлечённости ✅
    Elementor Pro добавляет 200+ КБ JS Медленная загрузка ✅
    jQuery legacy Устаревший стек ✅
    11 плагинов, 5 избыточных Раздутый код ✅
    Meta description отключён -30% CTR в поиске ✅
    robots.txt Disallow: /*?* Блокирует пагинацию ✅
    Schema.org UsedCondition Пессимизация Google ✅
    Email vandex Потеря лидов ✅
    2 разных телефона Путаница ✅

    2. Архитектурные ограничения WordPress¶

    Фундаментальные проблемы: - ❌ WordPress не headless-friendly — монолитная CMS, фронт и бэк связаны - ❌ WooCommerce — сложно масштабировать, тяжёлый - ❌ Elementor — генерирует «мусорный» HTML, тяжело кастомизировать - ❌ PHP runtime — нужен постоянный сервер (не serverless) - ❌ MySQL — нужен managed-хостинг, бэкапы - ❌ Плагины — конфликты, безопасность, обновления - ❌ Нет встроенной поддержки AI-агентов — нужны костыли

    Для AI-агента: - ❌ WordPress API сложный и нестабильный - ❌ WooCommerce REST API медленный - ❌ Нет нативного MCP (Model Context Protocol) или аналога - ❌ Управление контентом через Web UI, не через API-first

    3. Чего НЕ МОЖЕТ текущий стек¶

    Требование Можно ли на WP/WC
    Управление через AI-агент 🟡 через костыли
    Двусторонний обмен данными в real-time 🔴 нет
    Serverless хостинг 🔴 нет
    Edge computing 🔴 нет
    Мгновенная инвалидация кэша 🟡 сложно
    A/B тесты без плагинов 🔴 нет
    Встроенный image optimization 🟡 через плагины
    Авто-генерация контента 🔴 нет
    Чат-бот с доступом к каталогу 🟡 сложно
    Многоязычность из коробки 🟡 через WPML
    CRM из коробки 🔴 нет
    Mobile-first PWA 🟡 сложно
    Подписки / recurring payments 🟡 через плагины
    Бронирования туров (календарь) 🔴 нет

    4. Ограничения двух бизнесов на одном сайте¶

    Текущий подход — два раздела на одном домене: - Магазин (товары) - Турагентство (заглушка)

    Проблемы: - ❌ Разная аудитория (B2C покупатели vs B2C путешественники) - ❌ Разная воронка (купить vs забронировать тур) - ❌ Разный контент (товары vs туры с датами) - ❌ Разные интеграции (РобоКасса vs платёжные системы туров) - ❌ SEO конкуренция внутри одного сайта

    ✅ СИНТЕЗ — каким должен быть новый стек¶

    Требования к новому стеку¶

    1. Архитектурные требования¶

    • ✅ API-first — всё управляется через API
    • ✅ Headless / Decoupled — разделение фронта и бэка
    • ✅ Serverless-ready — масштабирование без DevOps
    • ✅ Edge-rendered — быстрая загрузка из любой точки
    • ✅ TypeScript everywhere — типобезопасность

    2. AI-агент требования (КРИТИЧНО)¶

    • ✅ MCP-совместимость (Model Context Protocol) или нативная поддержка AI-агентов
    • ✅ REST/GraphQL API для всех сущностей (товары, заказы, бронирования)
    • ✅ Webhooks для событий (новый заказ, бронь, отзыв)
    • ✅ WebSocket / SSE для real-time обновлений
    • ✅ SDK для Python/Node — для написания агента
    • ✅ Vector DB — для семантического поиска по каталогу
    • ✅ Function calling — для вызова бизнес-логики

    3. E-commerce требования¶

    • ✅ Продукты с вариациями (размер, цвет)
    • ✅ Категории с вложенностью
    • ✅ Фильтрация и поиск
    • ✅ Корзина и чекаут
    • ✅ Онлайн-платежи (РобоКасса, ЮKassa, СБП)
    • ✅ Доставка (СДЭК, Почта России, самовывоз)
    • ✅ Отзывы и рейтинги
    • ✅ Email-уведомления
    • ✅ Инвентаризация
    • ✅ Скидки и промокоды
    • ✅ Аналитика продаж

    4. Турагентство требования¶

    • ✅ Туры с датами и местами
    • ✅ Бронирование (календарь)
    • ✅ Разные пакеты (эконом, стандарт, премиум)
    • ✅ Гиды (множественные исполнители)
    • ✅ Локации (Байкал, Саяны, Бурятия)
    • ✅ Сезонность (только зима? или круглый год?)
    • ✅ Отзывы на туры
    • ✅ Галерея фото/видео
    • ✅ Программа тура по дням
    • ✅ Включённые услуги (трансфер, питание, проживание)

    5. UX/UI требования¶

    • ✅ Mobile-first
    • ✅ Core Web Vitals в зелёной зоне
    • ✅ PWA (установка на телефон)
    • ✅ Оффлайн-режим
    • ✅ Push-уведомления
    • ✅ Видеофон
    • ✅ 3D-обзор товара
    • ✅ AR (примерка на ботинок)

    6. SEO требования¶

    • ✅ SSR/SSG из коробки
    • ✅ Schema.org автоматически
    • ✅ Sitemap и robots.txt
    • ✅ Многоязычность (i18n)
    • ✅ Canonical URLs
    • ✅ Open Graph + Twitter Cards
    • ✅ Hreflang

    7. Маркетинг требования¶

    • ✅ Программа лояльности
    • ✅ Реферальная программа
    • ✅ UGC (отзывы с фото)
    • ✅ Интеграция с VK, Telegram

    • ✅ Heatmap (Hotjar/Clarity)

    🔧 Варианты стеков (для обсуждения)¶

    Ниже — современные стеки, ранжированные по соответствию требованиям ГРОМ38

    Вариант 1: Next.js + Medusa.js + AI-агент (PYTHON)¶

    Самый рекомендуемый для e-commerce + AI

    Слой Технология
    Фронт (главный сайт) Next.js 15 (App Router)
    UI компоненты shadcn/ui + Tailwind CSS 4
    E-commerce бэк Medusa.js 2.0 (Node.js)
    Турагентство Strapi 5 (Node.js)
    AI-агент Python + LangChain + MCP
    БД магазина PostgreSQL 16
    БД туров PostgreSQL 16 (отдельная)
    Vector DB Qdrant или pgvector
    Хостинг магазина Vercel (фронт) + Railway/Render (бэк)
    Хостинг туров Vercel (фронт) + Railway (бэк)
    Платежи РобоКасса / ЮKassa / СБП (виджеты)
    Email Resend + React Email
    Аналитика PostHog (self-hosted)
    Мониторинг Sentry

    Плюсы: - ✅ API-first — идеально для AI-агента - ✅ Headless — максимальная гибкость - ✅ Medusa.js специально для e-commerce - ✅ Strapi для контента (туры, блог) - ✅ TypeScript из коробки - ✅ SEO отличное (SSR/SSG) - ✅ Производительность (Edge) - ✅ MCP-совместимость

    Минусы: - 🟡 Нужны 2 бэкенда (магазин + туры) - 🟡 Дороже WP - 🟡 Дольше разработка - 🟡 Команда должна знать Next.js + Medusa + Strapi

    🎯 Рекомендация (для обсуждения)¶

    Главный фаворит: Вариант 1 — Next.js + Medusa.js + Strapi + AI-агент¶

    Почему: 1. ✅ API-first — идеально для AI-агента 2. ✅ Medusa.js — лучший open-source headless e-commerce для Node.js 3. ✅ Strapi — зрелая CMS для туров и контента 4. ✅ Next.js 15 — самый популярный React-фреймворк 2026 5. ✅ TypeScript everywhere — типобезопасность 6. ✅ Русские платежи легко — через виджеты РобоКасса 7. ✅ MCP-ready — для AI-агента 8. ✅ Масштабируется — от стартапа до enterprise

    Альтернатива (если нужен максимальный перформанс): Вариант 4 — Astro + Medusa + Directus¶

    Почему: - Astro 5 — самый быстрый фреймворк 2026 - Directus — мощнее Strapi для туров с календарём - Идеально для контент-тяжёлых сайтов

    Запасной вариант (если нужен быстрый старт): Вариант 2 — Next.js + Shopify Plus¶

    Почему: - Shopify — проверенная платформа - Встроенные платежи, доставка, налоги - Быстрый запуск

    Но: - ❌ Дорого - ❌ Выходит из РФ - ❌ Турагентство всё равно отдельно
